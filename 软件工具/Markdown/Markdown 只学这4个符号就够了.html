Markdown 只学这 4 个符号就够了

三金 三金 金途马语 2026年5月1日 18:15 广东

上一篇我们讲了 Inbox 怎么用。

核心不是把东西丢进去，而是让信息继续往后走：

Inbox → Notes → Topics → 输出

但很多新手真的开始写笔记时，又会卡在另一个地方：

Markdown 到底要不要学？

一看到各种语法表，标题、引用、代码块、表格、图片、任务列表、脚注……

很容易产生一种错觉：

是不是要先把 Markdown 学完，才能开始用 Obsidian？

我的建议很简单：

不用。

新手阶段，Markdown 不应该变成新的门槛。

你只需要先学 4 个最常用的符号：

#   标题
-   列表
**  加粗
[]() 链接

这 4 个，已经足够你写清楚 80% 的日常笔记。

一、先别把 Markdown 当成一门技术

很多人对 Markdown 的第一印象是：

这是不是程序员才用的东西？

其实不是。

Markdown 的本质，是一种很轻的写作格式。

它不是为了让文章变复杂，而是为了让你用很少的符号，把文章结构写清楚。

比如你在普通文档里写一篇笔记，可能会不断调整字号、颜色、缩进、样式。

但在 Markdown 里，你只需要用一个符号告诉系统：

这是一段标题
这是一个列表
这句话很重要
这里有一个链接

它关心的不是“好不好看”。

它先帮你解决一个更基础的问题：

这篇笔记有没有结构？

对 Obsidian 新手来说，这就够了。

因为前期最重要的不是排版漂亮，而是把想法写下来，并且以后还能看懂。

二、第一个符号：#，用来写标题

第一个要学的是 #。

它用来表示标题。

# 一级标题
## 二级标题
### 三级标题

在 Obsidian 里，标题不只是为了变大变粗。

它还有一个很重要的作用：

帮你把一篇笔记分成几个部分。

比如你处理一条 Inbox 内容，可以这样写：

## 原始内容
这里放链接、摘录或截图说明。

## 我的判断
这条内容说明了什么问题？

## 可能用途
以后可以用在哪篇文章或哪个主题里？

你看，这篇笔记还没写多少内容，结构已经出来了。

新手最容易犯的错误，是一整篇笔记从头写到尾，中间没有标题。

当内容很短时还好。

但只要超过几百字，你自己以后再看，也会很累。

所以我建议你记住一个简单规则：

只要一篇笔记超过 300 字，就至少加 2 个小标题。

标题不是装饰。

标题是在帮未来的你快速找回重点。

三、第二个符号：-，用来写列表

第二个要学的是 -。

它用来写列表。

- 一个想法
- 一个案例
- 一个问题
- 一个待处理动作

为什么列表很重要？

因为很多笔记，一开始并不需要写成完整文章。

它们更像一堆还没整理好的想法。

这时候不要逼自己写成大段文字。

先用列表接住就行。

比如你看到一篇文章，觉得以后可以用来做选题，不需要马上写完整笔记。

可以先这样写：

## 我的判断
- 这篇文章讲的是新手为什么容易放弃知识管理
- 核心问题不是工具难，而是流程太复杂
- 可以用来写 Day10：新手别一开始就装插件

列表的好处是：

它降低了开始写的压力。

很多人笔记写不下去，不是因为没想法。

而是因为一开始就想写得很完整。

先写列表，就像跑步前先慢跑热身。

不要一上来就冲配速。

先让系统动起来。

四、第三个符号：**，用来标重点

第三个要学的是 **。

它用来加粗。

这是普通内容。

**这是重点内容。**

但加粗不要乱用。

如果一段话里到处都是重点，那就等于没有重点。

我建议新手只在 3 种地方加粗：

结论
判断
行动

比如：

**Inbox 不是仓库，而是临时中转站。**

这就是结论。

**新手阶段，先追求笔记能流动，不要追求系统完美。**

这就是判断。

**今天只做一件事：把最近 3 条收藏补一句自己的判断。**

这就是行动。

加粗的作用，不是让文章看起来更花。

而是让读者扫一眼，就能抓住你真正想表达的东西。

对你自己也是一样。

以后回看笔记时，粗体部分就是这篇笔记的骨架。

五、第四个符号：，用来放链接

第四个要学的是链接。

写法是：

[链接文字](https://example.com)

很多新手保存资料时，会直接丢一串网址。

这样当然也能保存。

但过几天再看，你很可能已经不知道这个链接是什么了。

所以我更建议你这样写：

[Obsidian 官方 Markdown 基础语法](https://obsidian.md/help/syntax)

链接文字要写清楚。

不要只写：

[点击这里](https://example.com)

因为“点击这里”对未来的你没有任何信息量。

更好的方式是：

[Markdown Guide：基础语法说明](https://www.markdownguide.org/basic-syntax/)

也就是说，链接不是为了保存网址。

链接是为了保存上下文。

这和上一篇讲 Inbox 是同一个逻辑：

不要只保存东西，要保存你为什么保存它。

六、这 4 个符号，怎么组合成一条有用笔记？

现在把 4 个符号放在一起。

你可以得到一个非常简单的笔记模板：

## 原始内容
[文章标题](文章链接)

## 我的判断
- 这篇内容解决了什么问题
- 它和我正在关注的哪个主题有关
- 后面可能用在哪篇文章里

## 关键结论
**用一句话写下你自己的判断。**

## 下一步
- 删除
- 移到 Notes
- 连接到某个 Topic

你会发现，这个模板一点都不复杂。

但它已经完成了知识管理里最关键的几件事：

保留来源
写下判断
拆成结构
指向下一步

这比单纯收藏一篇文章有用得多。

因为它已经开始从“资料”变成“你的笔记”。

七、新手最容易踩的 3 个坑

第一个坑：想一次学完所有语法。

真的没必要。

表格、脚注、代码块、任务列表，以后用到再学。

你不是来考 Markdown 的。

你是来写笔记、做输出的。

第二个坑：太纠结格式。

标题用几级、列表缩进几格、加粗放在哪里，这些都可以慢慢统一。

前期最重要的是先写起来。

第三个坑：把 Markdown 当成排版工具。

它当然能排版。

但对 Obsidian 新手来说，它更重要的价值是：

帮助你把想法结构化。

当你的想法能被结构化，后面才容易连接、复用和输出。

最后

Markdown 只学这 4 个符号就够了：

#      写标题，搭结构
-      写列表，接住想法
**     加粗，标出重点
[]()   链接，保存上下文

不要把 Markdown 学成一门考试。

它只是 Obsidian 里的基础写作工具。

新手阶段，你只要能做到这 4 件事：

写出标题
列出要点
标出判断
保存来源

你的笔记就已经比单纯收藏强很多了。

今天不用学更多。

打开 Obsidian，新建一条笔记，用这 4 个符号写下最近保存的一篇文章。

只写 5 分钟就够。

先让笔记跑起来。
