01
基础概念说明

先说清楚两个概念
Skill（在部分 Trae CN 版本界面也可能显示为 Rules/规则） 是你教给 Trae 的"固定套路"。
比如你每次代码审查都要检查：可读性、正确性、安全性。有了 Skill，你只需要说"帮我代码审查"，Trae 就自动按这 3 步走，不用每次重新描述。
帮我做工作相关的事情
MCP 是让 Trae 连接外部工具的接口。
Trae 本身只能读写代码、执行命令。要查数据库、控制浏览器、调第三方 API，就需要对应的 MCP Server。
怎么选：
  • 任务只需要 AI 思考和生成 → Skill
MCP
  • 任务需要连接外部系统 → MCP

02
先跑通一个 Skill

以下操作均在 Trae CN（cn.trae.ai 下载的桌面版）中完成。


步骤 1：在 Trae CN 里新建技能
设置-创建技能-保存
  1. 打开 Trae CN
  2. 点击左下角（齿轮图标）
  3. 找到对应设置项
  4. 点击新建
  5. 填写名称和描述


步骤 2：填入内容并保存
说明： 以下内容粘贴到 Skill 的正文编辑区。frontmatter 中的 name/description 与界面字段保持一致即可。
---name: hello-skilldescription: 测试 Skill 是否工作version: 1.0.0---# Hello Skill## 触发条件当用户说"测试 skill"或"hello skill"时触发## 执行步骤1. 回复："✅ Hello！Skill 已成功触发！"2. 回复："配置正确，可以开始创建更多实用的 Skill 了。"
点击保存。


步骤 3：在 Trae CN 对话框测试
切换到对话框，输入触发词，看到对应回执就成功了。
✅ Hello！Skill 已成功触发！
没触发？按顺序排查：

  1. 确认 Skill 状态是"已启用"（Skills 列表里有开关）
  2. frontmatter 格式是否正确（包裹，冒号后有空格）
  3. 完全退出 Trae CN 重新打开
03
用提示词让 Trae CN 帮你生成 Skill（推荐方式）

你不需要手写 Skill 文件，直接在 Trae CN 对话框让 AI 生成。
通用提示词模板
帮我创建一个 Skill，用于 [你要解决的事情]。要求：1. 触发词：[词1]、[词2]2. 执行步骤：[步骤1] → [步骤2] → [步骤3]3. 输出格式：[字段1]、[字段2]请输出完整的 SKILL.md 内容。
Trae CN 返回内容后，直接复制到 Skills 新建面板保存即可。

04
个可直接用的 Skill 示例



写周报（普通人场景）
提示词：
帮我创建一个"写周报"Skill：- 触发词：写周报、本周总结- 先询问我本周做了什么（3-5 条）- 自动归纳问题与风险- 生成下周计划（3 条）- 输出格式：本周完成、问题与风险、下周计划、需要支持
或直接复制这个模板：
---name: weekly-reportdescription: 自动生成结构化工作周报version: 1.0.0---# weekly-report## 触发条件当用户说"写周报"、"本周总结"时触发## 执行步骤1. 询问用户：本周做了哪些事（3-5 条）2. 识别每条事项的结果和价值3. 归纳遇到的问题与风险4. 生成下周计划（3 条）5. 输出标准格式周报## 输出格式- 本周完成- 问题与风险- 下周计划- 需要支持


整理会议纪要（普通人场景）
提示词：
帮我创建一个"会议纪要整理"Skill：- 触发词：整理会议纪要、会议记录转纪要- 读取我粘贴的会议内容- 提取：关键结论、待办事项（Owner/DDL）、风险与依赖- 输出结构化纪要

或直接复制：
---name: meeting-minutesdescription: 把会议记录整理成可执行纪要version: 1.0.0---# meeting-minutes## 触发条件当用户说"整理会议纪要"、"会议记录转纪要"时触发## 执行步骤1. 读取原始会议内容2. 提取关键结论（决策项）3. 提取待办事项（负责人 + 截止时间）4. 提取风险与依赖5. 输出结构化纪要## 输出格式- 会议主题- 关键结论- 待办清单（Owner/DDL）- 风险与依赖


文章大纲生成（普通人场景）
提示词：
帮我创建一个"文章大纲生成"Skill：- 触发词：写文章大纲、帮我列提纲- 先确认目标读者和文章目标- 给出 3 个可选标题- 生成 6-8 段大纲，每段给出要点



代码审查（程序员场景）
提示词：
帮我创建一个"代码审查"Skill：- 触发词：帮我代码审查、review this code- 审查维度：可读性、正确性、安全性- 输出格式：严重问题（必须改）、重要问题（建议改）、可优化项
或直接复制：
---name: code-review-litedescription: 轻量代码审查（可读性/正确性/安全性）version: 1.0.0---# code-review-lite## 触发条件当用户说"帮我代码审查"、"review this code"时触发## 执行步骤1. 先回复："已进入 code-review-lite 流程"2. 读取本次改动3. 检查可读性4. 检查正确性5. 检查安全风险6. 输出问题清单和修改建议## 输出格式- 严重问题（必须改）- 重要问题（建议改）- 可优化项（锦上添花）



单元测试生成（程序员场景）
提示词：
帮我创建一个"单测生成"Skill：- 触发词：写单测、生成单元测试- 识别函数输入输出- 生成正常路径、边界条件、异常路径测试- 输出完整测试代码

05
个技巧

  • 触发词要短且明确：短触发词比长描述好得多
  • 执行步骤要动词化：用动作指令，不要写成描述性语句
  • 第一步加固定回执：在执行步骤第一条写确认内容，方便确认是否触发
先回复："已进入 xxx 流程"

06
MCP：现在需要了解什么（Trae CN）

Trae CN 的 MCP 能力在持续完善中，建议先用好 Skill，同时跟进官方更新（参考 Trae MCP 文档）。


 MCP（立刻能用）
下面给你 2 类测试方案：
  • A 类：在 Trae CN 自带 MCP 市场直接添加（推荐）
（大多数需要收费，可以按提示注册给钱添加）
  • B 类：手动 JSON 配置（通用兜底）
如果你要接官方开源 MCP Server或者自己研发的MCP，也可以粘贴 JSON：
{"mcpServers": {"hello-mcp-public": {"url": "http://47.106.195.239:8000/mcp"        }    }}



配置位置（Trae CN）
  1. 打开 Trae CN
  2. 点击设置 → 对应配置页
  3. 点击添加
优先在市场直接添加；没有再用 JSON 手动配置
添加后重启 Trae CN 生效


验证是否成功
配置后在对话框输入验证指令，Trae CN 会列出已加载的 MCP Server 和可用功能。
