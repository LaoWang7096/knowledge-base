AGENTS.md 实操指南：给 Codex 定制专属项目说明书

[AGENTS.md]可以理解为 Codex 的项目说明书。每次 Codex 启动工作前，都会自动读取这类文件，将内部规则、开发约定、测试指令、代码规范、安全要求等纳入运行上下文。


无需反复口头叮嘱操作要求，诸如改完代码执行测试、严控新增依赖、指定目录专属命令等约束，都会自动生效。依据 OpenAI 官方文档说明，Codex 会在任务启动阶段加载指令，全局规则与项目规则相互叠加生效。

 

一、指令加载顺序

Codex 会按照目录层级依次检索指令文件，优先级逐层递进：

  • 全局指令：默认读取~/.codex 目录下配置

  • 项目指令：从项目根目录逐级检索至当前工作目录

规则优先级遵循就近覆盖原则，越贴近当前运行目录的规则，生效权重越高。

常用两类指令文件：

  • AGENTS.md：存放长期稳定通用规则

  • AGENTS.override.md：用于临时覆盖原有规则

同一目录下，Codex 会优先加载 override 覆盖文件。

 

二、创建全局 [AGENTS.md]

全局配置用于定义个人通用开发习惯，包含测试方式、包管理器偏好、依赖新增校验等通用约束。

执行命令创建目录：

mkdir -p ~/.codex


编写全局规则文件~/.codex/AGENTS.md

## 通用工作约定- 修改 JavaScript 或 TypeScript 文件后，运行对应测试。- 优先使用项目已有的包管理器。- 新增生产依赖前先说明原因。- 不要重写无关文件。

配置完成后，所有项目启动 Codex 都会默认加载这套全局规范。可调用指令让 Codex 汇总已加载规则，校验文件是否正常生效。

 

三、创建项目级 [AGENTS.md](AGENTS.md)

放置在项目根目录，专门适配当前仓库业务专属规范，适配团队协同开发场景。

项目根目录AGENTS.md 参考示例

## 项目规范- 修改后端接口后，运行 `npm run test:api`。- 修改 UI 后，运行 `npm run lint`。- 公共工具函数需要补充文档。- 数据库迁移文件不要手动改历史记录。

项目规则会与全局规则合并生效，团队成员共用同一份约束标准，统一 Codex 行为逻辑。

 

四、子目录精细化定制规则

多服务、多模块拆分的仓库，可在子目录单独配置规则，实现局部差异化管控。

目录结构参考

my-repo/  AGENTS.md  services/    payments/      AGENTS.override.md    search/      AGENTS.md


支付服务子目录services/payments/AGENTS.override.md

## 支付服务规则- 测试支付模块时运行 `make test-payments`。- 不要改动密钥轮换逻辑，除非任务明确要求。- 涉及账务金额时，优先检查精度和舍入规则。

在该子目录启动 Codex，会依次加载全局规则、仓库根规则、子目录覆盖规则，细分规则优先级更高。

 

五、自定义识别文件名

若团队已有固定规范文档，可在配置中自定义文件名称，让 Codex 主动识别读取。

修改配置文件~/.codex/config.toml

project_doc_fallback_filenames = ["TEAM_GUIDE.md", ".agents.md"]project_doc_max_bytes = 65536

默认指令读取上限为 32 KiB，project_doc_max_bytes 参数可扩容限制；内容较多时，也可拆分规则至对应层级目录文件中。

 

六、校验规则是否正常生效

通过指令查看当前加载的全部指令来源，验证配置状态。

基础校验命令

bashcodex --ask-for-approval never "总结当前加载的指令"

指定目录针对性测试

bashcodex --cd services/payments --ask-for-approval never "列出当前生效的指令来源"

输出结果包含全局、项目根、子目录文件记录，即代表配置正常生效。

 

七、常见问题排查

- 规则未加载：确认处于项目正确目录，排查文件是否为空文件
- 规则匹配异常：检查上级目录、全局目录是否存在 override 覆盖文件
- 自定义文件名失效：核对配置名称拼写，重启 Codex 新建会话重试
- 规则内容截断：上调读取字节上限，拆分大容量规则至对应子目录

Codex 每次全新会话都会重新构建指令体系，无需手动清理缓存。

 

实用编写思路

精简凝练为宜，将日常高频重复的操作要求写入文档即可。

## 开发流程- 开始改动前，先阅读相关模块已有实现。- 保持改动范围小，不做无关重构。- 修改公共 API 时，更新测试和文档。## 验证命令- 前端：`npm run lint && npm test`- 后端：`pytest`- 类型检查：`npm run typecheck`## 安全边界- 不提交 `.env`、密钥或访问令牌。- 不在日志中打印用户隐私数据。

 

文末总结

[AGENTS.md]是绑定 Codex 的长效项目上下文。
全局文件定义个人开发偏好，项目根文件统一仓库规范，子目录适配局部特殊逻辑，临时调整需求可使用AGENTS.override.md 快速覆盖。
